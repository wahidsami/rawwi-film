# أوامر التحليل المرسلة إلى OpenAI وكيف يعمل المسار

هذا القسم يلخص **المسار الكامل للتحليل** و**نص الأوامر (system/user)** التي نرسلها إلى OpenAI.

---

## 1. كيف يعمل التحليل (المسار)

1. **إنشاء المهمة (Edge Function `tasks`)**  
   المستخدم يضغط "بدء التحليل الذكي". الـ Edge Function تقسم النص إلى **قطع (chunks)** (حجم ~12,000 حرف، تداخل 800). تُنشأ سجلات في `analysis_jobs` و `analysis_chunks`.

2. **المعالجة لكل قطعة (Worker)**  
   لكل قطعة ذات حالة `pending`:
   - **معجم (Lexicon):** مطابقة ألفاظ من جدول `slang_lexicon` (نص + حدود كلمة/عبارة). أي تطابق → إدراج كـ finding إلزامي (`source = lexicon_mandatory`).
   - **Router (اختياري):** إذا لم يكن `WORKER_HIGH_RECALL=true`، نرسل القطعة + قائمة المواد إلى نموذج واحد ليختار **المواد الأكثر صلة** (حتى 8). الناتج: `candidate_articles`.
   - **Judge متعدد المسارات (Multi-Pass Judge):** نمرّر القطعة + المواد (كل المواد إذا HIGH_RECALL، وإلا المواد المختارة من Router) إلى **عدة مسارات متوازية**، كل مسار له أمر منفصل:
     - مسار 0: **Glossary** — ألفاظ المعجم المحظورة (من DB).
     - مسار 1: **Insults** — ألفاظ مسيئة وشتائم.
     - مسار 2: **Violence** — عنف وتهديدات.
     - مسار 3: **Sexual content** — محتوى جنسي.
     - مسار 4: **Drugs & alcohol** — مخدرات وكحول.
     - مسار 5: **Discrimination & incitement** — تمييز وتحريض.
     - مسار 6: **National security** — أمن وطني وثوابت حكم.
     - مسار 7: **Extremism & banned groups** — تطرف وجماعات محظورة.
     - مسار 8: **Misinformation** — معلومات مضللة وشائعات.
     - مسار 9: **International relations** — علاقات دولية.
   - كل مسار يرجع `findings[]` مع `article_id`, `atom_id`, `evidence_snippet`, `location` (start/end offset). تُدمج النتائج وتُزال التكرارات (نفس المادة + موقع + دليل).
   - **Hybrid / Auditor:** إذا كان المحرك `hybrid`، تُجمَّع الـ findings في "مرشحات قانونية (canonical candidates)" وتُرسل مع **نص القطعة الكامل** إلى **Deep Auditor**. المدقق يقرأ النص والسياق ويصدر لكل مرشح: `final_ruling` (violation / needs_review / context_ok)، `primary_article_id`, `related_article_ids`, و **`rationale_ar`** (لماذا اعتُبرت مخالفة أو لا).
   - **Rationale-only pass:** إذا بقي أي finding بدون تعليل (أو التعليل افتراضي)، يُستدعى نموذج إضافي مرة واحدة لملء `rationale_ar` فقط.
   - تُحفظ كل الـ findings في `analysis_findings` مع `rationale_ar` و `location.v3`.

3. **التجميع (Aggregation)**  
   عندما لا تبقى قطع في حالة `pending` أو `judging`:
   - تُحمَّل كل الـ findings الخاصة بالـ job.
   - **تجميع حسب التداخل (Overlap clustering):** findings التي تشترك في نفس المقطع النصي (نفس الموقع تقريباً) تُجمَّع في "canonical finding" واحد. خيار المستخدم يتحكم هنا:
     - **بطاقة واحدة لكل موقع (موصى به):** دمج إذا تداخل المقطع ≥ 85% (نفس الحادثة، مواد متعددة على بطاقة واحدة).
     - **بطاقة لكل ظهور:** لا دمج؛ كل finding = بطاقة منفصلة.
   - **Report gate:** إذا كان نص "لماذا اعتُبرت مخالفة" يوضح أن المحتوى **لا يعد مخالفة** (مثل "السياق مقبول"، "لا يعد مخالفة")، تُنقل البطاقة إلى قسم "ملاحظات خاصة" ولا تُحسب ضمن إجمالي المخالفات.
   - **Words to revisit:** استدعاء خفيف (Revisit Spotter) على النص الكامل + قائمة مصطلحات المعجم النشطة؛ الناتج قائمة "كلمات/عبارات للمراجعة" في التقرير فقط (لا تغيّر عدد المخالفات).
   - يُبنى `summary_json` و `report_html` ويُحدَّث `analysis_reports` وتُحدَّد حالة الـ job `completed`.

## 2. الأوامر المرسلة إلى OpenAI (Prompts)

### 2.1 Router (اختيار المواد المرتبطة بالقطعة)

- **الغرض:** اختيار حتى 8 مواد الأكثر صلة بمقطع النص.
- **النموذج:** عادة `gpt-4.1-mini`.
- **System message (ROUTER_SYSTEM_MSG):**
  - أنت مرشّح فقط؛ مهمتك اختيار المواد الأكثر صلة بمقطع النص من قائمة المواد المعطاة.
  - قاعدة إلزامية: إذا احتوى النص على سبّ، شتم، إهانة، إساءة قائمة على الجنس، عدائية لفظية أو تهديد → أضف المواد [4، 5، 7، 17].
  - قاعدة المعجم: إذا احتوى النص على أي لفظ من قائمة الألفاظ المحظورة (من DB) أو مشتقاتها → أضف المواد المرتبطة.
  - المخرجات: JSON فقط: `{ "candidate_articles": [ { "article_id", "confidence" } ], "notes_ar": "اختياري" }`.
- **User message:** قائمة المواد (المادة X: العنوان) ثم `---` ثم مقطع النص (حتى ~15,000 حرف) ثم: "أرجع JSON بقائمة candidate_articles فقط."

---

### 2.2 Judge (كاشف المخالفات) — النسخة الأحادية (JUDGE_SYSTEM_MSG)

تُستخدم هذه النسخة في الـ Edge أو عندما لا يُستخدم Multi-Pass. في التشغيل الحالي الـ Worker يستخدم **Multi-Pass** (أدنى).

- **الغرض:** كشف كل مخالفة في النص مع دليل وموقع ومادة/بند.
- **System message (مختصر):**
  - وضع التشغيل: MAXIMUM DETECTION MODE — كشف كل محتوى منتهك، لا تبرير بالسياق الدرامي.
  - المرحلة 1: فحص الألفاظ المحظورة من المعجم؛ اللفظ موجود = مخالفة فورية، confidence = 1.0.
  - المرحلة 2: فحص المواد المعطاة كلمة بكلمة؛ أي محتوى يمكن أن ينتهك المادة → أخرج مخالفة.
  - أمثلة: عنف (ضرب، قتل، أسلحة، تهديد)، إهانة/سب، محتوى جنسي، مخدرات/كحول، تمييز، تحريض.
  - استثناءات: عناوين مشاهد، مدد زمنية، إشارات تقنية بحتة.
  - قواعد تقنية: استخدام `atom_id` بصيغة رقم-رقم (مثل 4-1)، كل finding يحتوي `evidence_snippet` و `location` (start_offset, end_offset).
  - المخرجات: JSON فقط: `{ "findings": [ { "article_id", "atom_id", "title_ar", "description_ar", "severity", "confidence", "is_interpretive", "evidence_snippet", "location": { "start_offset", "end_offset", "start_line", "end_line" } } ] }`.

---

### 2.3 Multi-Pass Judge (المسارات المتخصصة)

كل مسار له **نفس التذكير الأعلى** ثم مهمة محددة:

- **تذكير مشترك (MAX_DETECTION_NOTE):**  
  "وضع الكشف الأقصى: مهمتك كشف كل مخالفة. لا تتساهل. أي لفظ أو وصف يلامس المادة = أخرج مخالفة."

- **Pass 0 — Glossary:**  
  الألفاظ المحظورة من المعجم (قائمة من DB)؛ المواد المرتبطة. المهمة: ابحث عن أي لفظ من القائمة + مشتقاته؛ إذا وجدت اللفظ أخرج مخالفة. استثناء: عناوين مشاهد ومدد زمنية بحتة. المخرجات: نفس صيغة findings مع article_id/atom_id من المعجم.

- **Pass 1 — Insults:**  
  قائمة ألفاظ مسيئة (نصاب، حرامي، كذاب، …، عديم التربية، حمارة، فاشل، "العن أمك"، "موتو و خلصوني"، إلخ). اقرأ النص كلمة بكلمة؛ أي لفظ يمس الكرامة أو يحقر أو يهين = مخالفة. evidence_snippet = العبارة المهينة القصيرة فقط.

- **Pass 2 — Violence:**  
  أوصاف عنف (ضرب، صفع، ركل، قتل، أسلحة، تهديد، عنف أسري، ضرب الطفل، "اخذ الجزمة"، "اضربك"، "يدفع سامي بقوة فيقع"، إلخ). أي وصف لإيذاء جسدي أو تهديد = مخالفة.

- **Pass 3 — Sexual content:**  
  إيحاء جنسي، تلميح، غزل فاضح، وصف جسدي مثير، زنا، خيانة، قبلة، عناق حميمي، إلخ. أي محتوى جنسي أو إيحاء = مخالفة.

- **Pass 4 — Drugs & alcohol:**  
  مخدرات، حشيش، كحول، خمر، تعاطي، سكران، ترويج، حفلة شرب، إلخ. أي ذكر للمخدرات أو الكحول = مخالفة.

- **Pass 5 — Discrimination & incitement:**  
  تمييز عنصري/جندري/ديني، احتقار المرأة، "مكان البنت المطبخ"، تحريض، خطاب كراهية، تطرف، إلخ. أي تمييز أو تحريض = مخالفة.

- **Pass 6 — National security:**  
  المساس بالأمن الوطني، الإساءة للرموز، ثوابت الحكم، التحريض على قلب النظام، الإخلال بالنظام العام، إلخ.

- **Pass 7 — Extremism & banned groups:**  
  ترويج للجماعات المحظورة، إرهاب، رموز/شعارات، شرعنة، تطبيع.

- **Pass 8 — Misinformation:**  
  معلومات مضللة، شائعات، "أوامر سرية"، "حسابات وهمية"، تحريض، "الناس تصدق اللي يخوفها"، إلخ.

- **Pass 9 — International relations:**  
  الإساءة للدول/الشعوب، الإضرار بالمصالح، إثارة توترات، تشويه السياسة الخارجية.

في كل مسار: **المواد** = نصوص المواد والبنود من PolicyMap (يُحقن بناءً على Router أو كل المواد). المخرجات: `{ "findings": [ ... ] }` بنفس الصيغة الموحدة.

---

### 2.4 Deep Auditor (مدقق السياق)

- **الغرض:** إصدار حكم نهائي لكل "مرشح قانوني" (canonical candidate) مع ربط قانوني وتعليل عربي.
- **النموذج:** من الإعداد (مثل `gpt-4.1`).
- **System message (AUDITOR_SYSTEM_MSG):**
  - أنت مدقق امتثال؛ مهمتك إصدار حكم نهائي لكل finding مع ربط قانوني دقيق، وليس زيادة عدد المخالفات.
  - قواعد: نفس `canonical_finding_id` مرة واحدة فقط؛ مادة أساسية واحدة في `primary_article_id`؛ الباقي في `related_article_ids`.
  - `final_ruling`: واحدة فقط من `violation` | `needs_review` | `context_ok`.
  - **rationale_ar إلزامية:** اشرح أين يظهر المقتطف في النص (حوار، حلم، وصف)، ماذا يعني في السياق السردي، ولماذا اعتُبرت مخالفة (أو needs_review أو context_ok) وربطها بالمادة. إذا الحكم context_ok أو أن المحتوى لا يعد مخالفة، اذكر ذلك صراحة (مثل "السياق مقبول"، "لا يعد مخالفة") حتى تُصنَّف في التقرير كتنبيه وليس كمخالفة.
  - confidence_breakdown: lexical / context / policy (0..1). لا تخترع أدلة خارج النص.
  - المخرجات: JSON فقط: `{ "assessments": [ { "canonical_finding_id", "rationale_ar", "title_ar", "final_ruling", "pillar_id", "primary_article_id", "related_article_ids", "confidence", "confidence_breakdown", "severity" } ] }`.
- **User message:** "المرشحات القانونية canonical:" + (payload المرشحات) + "مقتطف النص الكامل:" + (نص القطعة حتى ~35,000 حرف). "أرجع JSON فقط. كل assessment يجب أن يحتوي حقل rationale_ar مملوءاً (جملة أو جملتان بالعربية: أين في النص، ماذا يعني في السياق، ولماذا اعتُبرت مخالفة أو تحتاج مراجعة)."

---

### 2.5 Rationale-only (ملء التعليل فقط)

- **الغرض:** ملء `rationale_ar` فقط للـ findings التي لم تحصل على تعليل من المدقق الرئيسي.
- **System message (RATIONALE_ONLY_SYSTEM_MSG):**
  - مهمتك الوحيدة: لكل عنصر في القائمة، اكتب جملة أو جملتين بالعربية في `rationale_ar`: (1) أين يظهر المقتطف، (2) ماذا يعني في السياق، (3) لماذا اعتُبر مخالفة أو تحتاج مراجعة وربطها بالمادة. إذا المحتوى لا يعد مخالفة (سياق مقبول، دون إيحاء)، اذكر ذلك صراحة.
  - المخرجات: `{"rationales":[{"canonical_finding_id":"...","rationale_ar":"..."}]}`. كل عنصر مُدخل له عنصر مُخرج بنفس الـ id وتعليل غير فارغ.
- **User message:** "اكتب rationale_ar لكل عنصر بالعربية (جملة أو جملتان). أرجع JSON فقط: …" + قائمة العناصر (لكل عنصر: canonical_finding_id، evidence_snippet، final_ruling، primary_article_id).

---

### 2.6 Revisit Spotter (كلمات/عبارات للمراجعة)

- **الغرض:** إدراج قسم "كلمات/عبارات للمراجعة" في التقرير فقط؛ لا يحكم على مخالفة.
- **النموذج:** عادة `gpt-4.1-mini`.
- **System message (REVISIT_SPOTTER_SYSTEM):**
  - أنت مكتشف كلمات وعبارات فقط. ابحث في النص عن كل ظهور للكلمات/العبارات المعطاة. لا تحكم على المحتوى (لا تقل مخالفة أو لا). سجّل كل موضع تظهر فيه إحدى الكلمات أو العبارات.
  - المخرجات: JSON فقط: `{ "mentions": [ { "term", "snippet", "start_offset", "end_offset" } ] }`. استخدم أرقام الموضع من بداية النص (أول حرف = 0).
- **User message:** القائمة: (مصطلحات المعجم النشطة، حتى 80) ثم `---` ثم النص (حتى ~28,000 حرف). "أرجع JSON فقط: { \"mentions\": [ ... ] }."

## ملاحظات للاستخدام

1. **التعبئة اليدوية:** في عمود "ماذا نبحث عنه (يدوياً)" اكتب وصفاً واضحاً لما يجب أن يثير مخالفة تحت هذا البند (أمثلة ألفاظ، أوصاف، سياقات، سلوكيات، إلخ).
2. **ربط المادة/البند:** بعد التعبئة يمكن استخدام هذا الملف كمرجع لـ:
   - تحسين أوامر الـ Judge أو الـ Auditor.
   - بناء "دليل فهم القطعة ثم الربط" (فهم القطعة chunk-by-chunk ثم اختيار المخالفات وفق المادة والبنود).
3. **لماذا اعتُبرت مخالفة:** الهدف لاحقاً أن يعيد التحليل تعليلاً يربط **فهم القطعة** + **المقتطف** + **المادة/البند** بشكل واضح بدلاً من جملة عامة أو تكرار المقتطف فقط.
